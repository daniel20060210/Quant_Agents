## 三、核心 Agent 设计

### 3.1 交易Agent（TradingAgent）

交易Agent 是整个系统的决策中心，负责每一笔交易的完整生命周期。

```
┌─────────────────────────────────────────────┐
│                TradingAgent                  │
├─────────────────────────────────────────────┤
│                                              │
│  每个交易周期：                                │
│                                              │
│  1. 感知 ─ 获取当前持仓、行情数据、账户状态       │
│       │                                      │
│  2. 决策 ─ 遍历 skills，匹配当前市场环境        │
│       │    skill.analyze(context) → Signal    │
│       │    (BUY / SELL / HOLD)               │
│       │                                      │
│  3. 风控 ─ RiskManager.check(order)          │
│       │    硬性规则：单笔亏损 ≤ 1%              │
│       │    计算：止损价 = 入场价 × (1 - 1%)     │
│       │                                      │
│  4. 执行 ─ Broker.execute(order)             │
│       │    模拟成交，记录持仓                   │
│       │                                      │
│  5. 复盘 ─ 收盘后检查当日盈亏                   │
│       │    更新连续亏损计数器                   │
│       │    触发反思（如果需要）                  │
│       │                                      │
│  6. 反思 ─ 连续3笔亏损 → Reflection.analyze()  │
│            分析失败原因                         │
│            修改/替换 skill 参数                 │
│            必要时请求脚本Agent写分析脚本          │
│                                              │
└─────────────────────────────────────────────┘
```

## 四、Skills 技能系统

Skills 是交易Agent 的策略知识库。每个 Skill 封装一种交易逻辑。

### 4.1 Skill 基类

```python
class BaseSkill(ABC):
    """所有交易技能的基类"""

    name: str                    # 技能名称
    version: str                 # 版本号（反思修改后会递增）
    performance: SkillPerformance # 该技能的历史表现统计

    @abstractmethod
    def analyze(self, context: MarketContext) -> Signal:
        """分析市场，返回交易信号"""
        ...

    @abstractmethod
    def get_stop_loss(self, entry_price: float, context: MarketContext) -> float:
        """计算止损价（不能超过入场价的1%）"""
        ...

    @abstractmethod
    def get_take_profit(self, entry_price: float, context: MarketContext) -> float:
        """计算止盈价（由AI自行决定）"""
        ...

    def modify(self, new_params: dict) -> None:
        """反思后修改技能参数，version+1"""
        ...
```

### 4.2 Skill 示例

```python
class TrendFollowingSkill(BaseSkill):
    """趋势跟踪：均线金叉买入，死叉卖出"""

    params = {
        "fast_ma": 5,
        "slow_ma": 20,
    }

    def analyze(self, ctx):
        fast = ctx.ma(self.params["fast_ma"])
        slow = ctx.ma(self.params["slow_ma"])
        if fast crosses above slow:
            return Signal.BUY
        elif fast crosses below slow:
            return Signal.SELL
        return Signal.HOLD

class MeanReversionSkill(BaseSkill):
    """均值回归：偏离均线超阈值则反向操作"""

    params = {
        "ma_period": 20,
        "threshold": 0.02,  # 偏离2%
    }

    def analyze(self, ctx):
        ma = ctx.ma(self.params["ma_period"])
        deviation = (ctx.current_price - ma) / ma
        if deviation > self.params["threshold"]:
            return Signal.SELL   # 超涨 → 做空/卖出
        elif deviation < -self.params["threshold"]:
            return Signal.BUY    # 超跌 → 做多/买入
        return Signal.HOLD
```

## 五、风控引擎（硬性规则）

风控引擎是系统的**硬约束**，不可被任何 Agent 绕过。

```python
class RiskManager:
    MAX_LOSS_PCT = 0.01          # 单笔最大亏损 1%（硬性规则）

    def validate_order(self, order: Order, context: Context) -> bool:
        """
        校验订单是否符合风控规则：
        1. 必须设置止损，且止损距离 ≤ 1%
        2. 单笔仓位不超过总资金的合理比例
        3. 同一时间只能持有一单
        """
        # 规则1：止损必须在入场价的 99% 以上（多头）或 101% 以下（空头）
        if order.direction == LONG:
            max_allowed_loss = order.entry_price * (1 - self.MAX_LOSS_PCT)
            if order.stop_loss is None or order.stop_loss < max_allowed_loss:
                # 强制修正止损
                order.stop_loss = max_allowed_loss
                log.warning(f"止损被风控强制修正为 {max_allowed_loss}")

        elif order.direction == SHORT:
            max_allowed_loss = order.entry_price * (1 + self.MAX_LOSS_PCT)
            if order.stop_loss is None or order.stop_loss > max_allowed_loss:
                order.stop_loss = max_allowed_loss

        return True  # 通过或修正后通过
```

## 六、反思机制

反思机制有**两个独立的触发条件**，满足任一即触发：

```
┌──────────────────────────────────────────────────────────┐
│                    反思触发条件（可配置）                    │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  条件A：连续亏损                                            │
│  ┌─────────────────────────────────────────────────┐    │
│  │  连续 N 笔交易亏损 → 触发反思                       │    │
│  │  默认值：N = 3                                    │    │
│  │  示例：第3笔连续亏损 → 立即触发                       │    │
│  └─────────────────────────────────────────────────┘    │
│                                                          │
│  条件B：滚动回撤                                           │
│  ┌─────────────────────────────────────────────────┐    │
│  │  最近 M 笔交易内，累计回撤 ≥ X% → 触发反思           │    │
│  │  默认值：M = 10，X% = 6%                          │    │
│  │  示例：最近10笔交易累计亏损达到6% → 立即触发           │    │
│  └─────────────────────────────────────────────────┘    │
│                                                          │
│  所有阈值 (N, M, X%) 均可人工修改                          │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### 6.3 反思执行

```
┌──────────────────────────────────────────────┐
│  触发反思后                                    │
├──────────────────────────────────────────────┤
│                                               │
│  1. 收集相关交易数据                             │
│     - 条件A触发：收集最近连续亏损的 N 笔           │
│     - 条件B触发：收集最近观察窗口的 M 笔           │
│     - 入场时间、价格、skill名称、市场环境          │
│                                               │
│  2. 分析失败原因                                │
│     - 趋势判断错误？                            │
│     - 入场/出场时机问题？                        │
│     - 止损设置合理性？                           │
│     - 市场环境是否适合当前skill？                 │
│                                               │
│  3. 生成改进方案                                │
│     - 调整 skill 参数（如均线周期、阈值）          │
│     - 添加/修改过滤条件                          │
│     - 降低低效 skill 的权重或禁用                 │
│     - 请求脚本Agent生成分析脚本深入诊断            │
│                                               │
│  4. 应用修改                                   │
│     - skill.version += 1                      │
│     - 重置计数器（条件A：loss_streak=0）          │
│     - 写入反思日志                              │
│                                               │
└──────────────────────────────────────────────┘
```

```python
class Reflection:
    def __init__(self, config: ReflectionConfig):
        self.config = config

    def should_trigger(self, agent: TradingAgent) -> Tuple[bool, str]:
        """判断是否触发反思，返回 (触发, 触发原因)"""

        # 条件A：连续亏损
        if agent.loss_streak >= self.config.max_consecutive_losses:
            return True, f"连续亏损{agent.loss_streak}笔（阈值={self.config.max_consecutive_losses}）"

        # 条件B：滚动回撤
        recent = agent.trade_history[-self.config.lookback_trades:]
        total_pnl = sum(t.pnl_pct for t in recent)
        if total_pnl <= -self.config.max_drawdown_pct:
            return True, f"最近{self.config.lookback_trades}笔回撤{abs(total_pnl)*100:.1f}%（阈值={self.config.max_drawdown_pct*100}%）"

        return False, ""

    def execute(self, agent: TradingAgent, trigger_reason: str) -> ReflectionResult:
        trades = self._collect_trades(agent)
        prompt = self._build_prompt(trades, trigger_reason)
        analysis = llm.analyze(prompt)

        for skill in agent.skills:
            if skill.name in analysis.affected_skills:
                skill.modify(analysis.new_params)

        # 重置计数器
        agent.loss_streak = 0

        return ReflectionResult(
            trigger=trigger_reason,
            reason=analysis.reason,
            modifications=analysis.modifications,
        )

    def update_config(self, **kwargs):
        """人工修改阈值"""
        for key, value in kwargs.items():
            if hasattr(self.config, key):
                setattr(self.config, key, value)
        log.info(f"反思阈值已更新: {self.config}")
```

router = APIRouter(prefix="/api/config", tags=["config"])

### 7.2 交易引擎（替代原 main 循环）

```python
# core/engine.py
class TradingEngine:
    """交易引擎：封装所有核心组件，由 FastAPI 驱动"""

    def __init__(self):
        self.skills = load_skills()
        self.risk_mgr = RiskManager()
        self.broker = Broker(initial_capital=100_000)
        self.script_team = ScriptAgentTeam(
            requirement_agent=RequirementAgent(),
            engineer_agent=EngineerAgent(),
            test_agent=TestAgent(),
        )
        self.reflection_config = ReflectionConfig(
            max_consecutive_losses=3,
            lookback_trades=10,
            max_drawdown_pct=0.06,
        )
        self.reflection = Reflection(self.reflection_config)
        self.agent = TradingAgent(
            skills=self.skills,
            broker=self.broker,
            risk_manager=self.risk_mgr,
            script_team=self.script_team,
            config=AgentConfig(),
        )
        self._running = False

    def start(self):
        self._running = True
        # 启动后台交易日循环（异步）
        asyncio.create_task(self._trading_loop())

    async def _trading_loop(self):
        """后台交易循环，按日期逐日推进"""
        for trade_date, market_data in load_daily_data():
            if not self._running:
                break
            context = MarketContext(date=trade_date, data=market_data)
            await self.process_day(context)

    async def process_day(self, context: MarketContext):
        """处理一个交易日"""
        # 1. 平仓检查
        if self.agent.has_position():
            signal = self.agent.evaluate_exit(context)
            if signal == Signal.EXIT:
                self.agent.close_position(context)
                triggered, reason = self.reflection.should_trigger(self.agent)
                if triggered:
                    self.reflection.execute(self.agent, reason)
                # WebSocket 推送交易事件
                await ws_broadcast("trade_closed", {...})

        # 2. 开仓判断
        else:
            signal, skill = self.agent.evaluate_entry(context)
            if signal in (Signal.BUY, Signal.SELL):
                order = self.agent.create_order(signal, skill, context)
                self.risk_mgr.validate_order(order, context)
                self.broker.execute(order)
                await ws_broadcast("trade_opened", {...})

        # 3. 收盘复盘
        if is_market_close(context):
            self.agent.daily_review()
            await ws_broadcast("daily_review", {...})
```

### 7.3 API 路由设计

```
GET    /api/dashboard/overview          # 账户概览：净值、收益率、持仓
GET    /api/dashboard/equity_curve      # 权益曲线数据
GET    /api/dashboard/trade_history     # 交易历史列表
GET    /api/dashboard/current_signal    # 当前信号（哪个skill、方向、置信度）

GET    /api/skills                      # 获取所有skill及其状态
PUT    /api/skills/{name}              # 启用/禁用/修改某个skill参数

GET    /api/config/reflection           # 获取反思阈值
PUT    /api/config/reflection           # 修改反思阈值（3/10/6%）

POST   /api/trading/pause               # 暂停交易
POST   /api/trading/resume              # 恢复交易
POST   /api/trading/force_close         # 强制平仓（人工干预）
POST   /api/trading/skip_next           # 跳过下一个交易信号

WS     /ws                             # WebSocket：实时推送
```

### 7.5 WebSocket 推送事件

| 事件 | 触发时机 | 前端用途 |
|------|---------|---------|
| `trade_opened` | 开仓成交 | 弹窗通知、更新持仓卡片 |
| `trade_closed` | 平仓成交 | 显示盈亏、更新交易历史 |
| `daily_review` | 每日收盘复盘后 | 刷新看板数据 |
| `reflection_triggered` | 反思触发 | 告警提示、展示分析结果 |
| `skill_modified` | Skill 参数被修改 | 刷新 Skill 列表 |
| `signal_generated` | 产生交易信号 | 展示信号详情 |
